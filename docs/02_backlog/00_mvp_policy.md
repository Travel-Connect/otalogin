# MVP方針

## 確定要件（ユーザー確認済み）

### セキュリティ方針

| 項目 | MVP方針 | 備考 |
|------|---------|------|
| パスワード暗号化（DB/Sheets） | **不要** | 平文保存でOK |
| UI表示 | `****` マスク + 目アイコン | 10秒で自動マスク復帰 |
| ログ出力 | PW/Token/RefreshToken出力禁止 | 絶対厳守 |

### マスタPW同期（F4）

- 同期対象: **今開いているチャネル（facility × channel）のみ**
- 同期前: **確認ダイアログ表示**
- 実行: **Enterキーで実行可能**

### Health Check（F6）

- 実行時刻: **05:00 JST（= 20:00 UTC）**
- 対象: **shared アカウントのみ**（override は対象外）

### Chrome拡張（F5）

- **別ウィンドウ/別インスタンス起動は NG**
- ユーザーが操作中のChromeウィンドウにタブ追加
- `sender.tab.windowId` を使用して同一ウィンドウ維持

## MVP対象外（将来オプション）

以下はMVPスコープ外とし、将来の拡張として検討:

| 項目 | 理由 |
|------|------|
| パスワード暗号化（DB保存時） | ユーザー確認済みで不要 |
| WebSocket（リアルタイム通知） | polling で十分 |
| 施設の新規登録UI | DBで直接登録 or スクリプト |
| 複数ユーザー権限管理 | MVP は 5名全員同権限 |
| ログイン履歴詳細表示 | Health Status のみで判断 |

### Phase 3: 拡張統合（優先度: 中）

1. ポータル → 拡張のメッセージング動作確認
2. 拡張の pending job ポーリング実装

1. e2e:mock 実装・安定化

3. e2e:pack で成果物作成
