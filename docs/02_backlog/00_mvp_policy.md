# MVP方針

最終更新: 2026-03-09

## 確定要件（ユーザー確認済み）

### セキュリティ方針

| 項目 | 方針 | 備考 |
|------|------|------|
| パスワード暗号化（DB保存時） | **実装済み** | AES-256-GCM |
| UI表示 | `****` マスク + 目アイコン | 10秒で自動マスク復帰 |
| ログ出力 | PW/Token/RefreshToken出力禁止 | 絶対厳守 |

### マスタPW同期（F4）

- 同期対象: **施設単位で全チャネル一括**
- 同期前: **確認ダイアログ表示**
- admin ロールのみ実行可能

### Health Check（F6）

- 実行時刻: **05:00 JST（= 20:00 UTC）**
- 対象: **shared アカウントのみ**（override は対象外）

### Chrome拡張（F5）

- **別ウィンドウ/別インスタンス起動は NG**
- ユーザーが操作中のChromeウィンドウにタブ追加
- `sender.tab.windowId` を使用して同一ウィンドウ維持

## 将来オプション

以下は現在のスコープ外とし、将来の拡張として検討:

| 項目 | 理由 |
|------|------|
| shared/override切替UI | 現状sharedで運用に問題なし |
| WebSocket（リアルタイム通知） | polling で十分 |
| 施設の新規登録UI | DBで直接登録 or スクリプト |
| 複数ユーザー権限管理 | admin/一般の2種類で十分 |
| ログイン履歴詳細表示 | Health Status のみで判断 |
